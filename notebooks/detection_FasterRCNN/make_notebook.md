## 🔧 Key Implementation Details

### **1. Differences from YOLOv8 Notebooks**

| Aspect | YOLOv8 | Faster R-CNN |
|--------|---------|--------------|
| **Framework** | Ultralytics (high-level API) | Native PyTorch + torchvision |
| **Data Loading** | YAML config + automatic loading | Custom dataloader with COCO/Pascal/YOLO support |
| **Training Loop** | Handled by Ultralytics | Manual implementation with AMP |
| **Evaluation** | Built-in `model.val()` | Custom mAP calculation |
| **Backbone Customization** | Modify C2f modules | Modify ResNet50 layer structure |
| **Mixed Precision** | Automatic | Manual with `GradScaler` |
| **Metrics Logging** | Automatic CSV generation | Manual CSV writing |

### **2. Critical Implementation Notes**

**Data Loader Integration:**
- Must import `create_faster_rcnn_dataloaders` from `src.data_processing.faster_rcnn_dataloader`
- Handles invalid bbox filtering automatically
- Supports multiple annotation formats (COCO, Pascal, YOLO)

**Mixed Precision Training:**
- Uses `torch.cuda.amp.autocast` and `GradScaler`
- Gradient clipping with `torch.nn.utils.clip_grad_norm_`
- Essential for stable training on T4 GPUs

**Fast mAP Evaluation:**
- Custom `_fast_evaluate()` method (< 5 seconds per epoch)
- Simplified IoU matching instead of full COCO evaluation
- Tracks both mAP@0.5 and mAP@0.5:0.95

**Model Customization:**
- V2: Wraps layer3 with Sequential containing custom conv blocks
- V3: Rebuilds layer3 with fewer bottleneck blocks
- Both preserve FPN compatibility

## ⚠️ Expected Challenges & Solutions

### **Challenge 1: Dataloader Complexity**

**Problem:**  
Faster R-CNN requires custom dataloader with proper annotation parsing, unlike YOLOv8's simple YAML config.

**Solution:**
- Import `create_faster_rcnn_dataloaders` from existing module
- Handle invalid bbox filtering gracefully
- Print skipped annotation counts for transparency

### **Challenge 2: Mixed Precision Training**

**Problem:**  
Manual AMP implementation requires careful gradient scaling and error handling.

**Solution:**
- Use `autocast(enabled=use_amp)` context manager
- Scale losses with `scaler.scale(losses).backward()`
- Unscale gradients before clipping: `scaler.unscale_(optimizer)`
- Skip batches with NaN/Inf losses

### **Challenge 3: Fast mAP Calculation**

**Problem:**  
Full COCO mAP evaluation is slow (> 30 seconds per epoch), making long training impractical.

**Solution:**
- Implement simplified IoU-based matching (< 5 seconds)
- Track both mAP@0.5 and mAP@0.5:0.95 approximately
- Accept slight accuracy trade-off for speed

### **Challenge 4: Backbone Customization Compatibility**

**Problem:**  
Modifying ResNet50 structure must maintain FPN access to layer outputs.

**Solution:**
